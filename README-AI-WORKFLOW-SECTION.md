## AI Development Workflow

This repository includes AI-specific development instructions to help coding assistants work consistently.

### Required reading before changes
Any AI working in this repository should read:
- all `coreIdentity` files
- the `productRoadmap` document
- `README.md`
- the existing changelog, if present

### Documentation requirements
After making changes, the AI should:
- update or create the changelog
- append a brief entry without deleting older entries
- include the AI/tool name in the changelog
- update `README.md`
- update any other relevant documentation

### Multi-task behavior
If given a list of tasks, the AI should try to complete as many as possible, preferably all of them.
