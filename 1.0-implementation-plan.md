# Astra/log 1.0 Implementation Plan (Updated)

This document outlines the final development push for the 1.0 release, integrating advanced window management, AI interaction refinements, and user onboarding.

---

## Phase 1: Workspace Architecture & Responsive Layout
*Goal: Enable full flexibility for different workflows (Logic vs. Design) and ensure stability across devices.*

### 1.1 Multi-Configuration Layout Engine
* **Dynamic Grid System**: Migrate from a standard flexbox layout to a CSS Grid or `react-resizable-panels` system in `App.tsx` to allow vertical and horizontal splitting.
* **Minimize & Reorganize**:
    * Implement "Minimize to Tray" for Terminal, Chat, and Scratch Pad. The Header and Preview remain as fixed constants.
    * Add a "Layout Switcher" to the Header with three presets:
        * **Standard**: (Current) Terminal Left, Preview Center, Chat/Scratch Right.
        * **Architect**: Terminal Bottom (Full Width), Preview Top-Left, Chat/Scratch Top-Right.
        * **Zen Focus**: Scratch Pad Center-Left, Preview Center-Right (Terminal/Chat minimized).

### 1.2 Responsive Adaptation
* **Mobile/Tablet Breakpoints**: When the viewport width is `< 1024px`, automatically collapse non-essential panels into a bottom drawer.
* **Touch Optimizations**: Increase the hit-box of the `GripVertical` and horizontal resize dividers for easier manipulation on touch devices.

---

## Phase 2: AI Interaction & Contextual UI
*Goal: Evolve the AI from a general assistant into a specialized technical consultant.*

### 2.1 AI Behavior Layer (Consultant Mode)
* **Prompt Engineering**: Update system instructions in `ChatPanel.tsx` to strictly follow the "Consultant" role—prioritizing architectural explanation and diagnostic commands over unsolicited code rewrites.
* **Concise Mode Toggle**: Add a header toggle to limit AI response length and force bulleted summaries for rapid reading.

### 2.2 Contextual & Suggested Response Buttons
* **AI Action Chips**: Parse assistant messages to display high-value follow-ups:
    * `[Expand]`: Asks for technical depth on the last response.
    * `[What's missing]`: Asks the AI to identify architectural gaps.
* **Suggested Responses**: For questions identified as having short, common answers, display quick-tap buttons:
    * Binary options: `Yes`, `No`.
    * Contextual options: `Proceed`, `Explain further`, `Cancel`.
* **Draft-to-Chat Integration**: Add a button to the `ScratchPad` to "Stage Notes for AI," prepending local notes to the next message for context.

---

## Phase 3: User Onboarding & Knowledge Base
*Goal: Reduce the learning curve for the "Transient Workspace" concept.*

### 3.1 First-Open Onboarding
* **Welcome Modal**: Implement a one-time splash screen that explains the "Zero Persistence" and "Consultant-Only" core principles.
* **Workspace Tour**: A brief, 3-step highlight overlaying the Terminal (Ingestion), Preview (Output), and Scratch Pad (Private Thoughts).

### 3.2 In-App Documentation Guide
* **Usage Guide**: A dedicated help section (accessible via a `?` icon in the header) containing:
    * **Keyboard Shortcuts**: Cheat sheet for `Ctrl + .` (Scratch Pad) and `Cmd + K` (Command Palette).
    * **Workflow Tips**: How to use the Scratch Pad to iterate on ideas before involving the AI.

---

## Phase 4: Final Polish & 1.0 Verification
* **Visual Diff for Snapshots**: Update the `ExportModal` "Compare" view to highlight additions/deletions between plan states.
* **Logo Animation**: Add a subtle SVG path animation to the `AstraLogLogo` to provide ambient feedback that the app is active.
* **Session Freshness Indicator**: Add a subtle timer near the "New Project" button showing how long the current transient session has been active.
* **Security Audit**: Verify `scanPackageJson` correctly identifies high-risk scripts in all ZIP uploads before `npm install`.
